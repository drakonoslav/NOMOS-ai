The schema I would use

type EntityCategory =
  | "substance"
  | "food"
  | "fluid"
  | "supplement"
  | "object"
  | "load"
  | "duration"
  | "distance"
  | "countable_item"
  | "event"
  | "anchor"
  | "unknown";

type TagProvenance =
  | "explicit"
  | "registry"
  | "normalized"
  | "inferred"
  | "fallback";

interface TagRecord {
  tag: string;                         // e.g. "carb", "fast", "liquid", "pre_lift"
  provenance: TagProvenance;          // where it came from
  confidence: number;                 // 0.0 to 1.0
  sourceRegistryId?: string | null;   // if registry-driven
}

interface NormalizationRecord {
  stage: string;                      // "raw", "unit_normalization", "entity_normalization", etc.
  before: string;
  after: string;
  reason: string;
}

interface MeasureRecord {
  amount: number;
  unitRaw: string;                    // e.g. "grams", "g", "hours"
  unitNormalized: string;             // e.g. "g", "hour"
  dimension:
    | "mass"
    | "volume"
    | "count"
    | "time"
    | "distance"
    | "energy"
    | "rate"
    | "unknown";
}

interface CanonicalEntity {
  id: string;

  rawText: string;
  normalizedText: string;

  labelRaw: string;                   // e.g. "cyclic dextrin"
  labelNormalized: string;            // e.g. "cyclic_dextrin"

  category: EntityCategory;
  categoryConfidence: number;         // 0.0 to 1.0

  measures: MeasureRecord[];          // supports one or more measures if needed

  tags: TagRecord[];                  // canonical semantic tags

  role:
    | "candidate_item"
    | "constraint_operand"
    | "state_fact"
    | "objective_operand"
    | "anchor"
    | "unknown";

  sourceRegistryId?: string | null;
  normalizationHistory: NormalizationRecord[];

  modifiers?: string[];               // e.g. "fast-digesting", "pre-lift"
  notes?: string[];
}


⸻

Why this schema is strong

It separates things that should not be conflated:

1. category

What kind of thing it broadly is.
	•	food
	•	supplement
	•	duration
	•	load

2. tags

What semantic properties matter for reasoning.
	•	carb
	•	fast
	•	slow
	•	protein
	•	fluid
	•	stimulant
	•	pre_lift

3. tagProvenance

Where the tag came from.
	•	explicit
	•	registry
	•	inferred
	•	fallback

4. confidence

How much NOMOS should trust that classification.

5. sourceRegistryId

Which registry entry anchored the normalization, if any.

6. normalizationHistory

How the raw text became the canonical entity.

That is the difference between “works sometimes” and “auditably works.”
