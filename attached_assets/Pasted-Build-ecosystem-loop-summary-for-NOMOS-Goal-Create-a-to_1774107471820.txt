Build ecosystem loop summary for NOMOS.

Goal:
Create a top-level reflective layer that summarizes the full governance ecosystem across time, including:

- what prediction patterns led to which governance choices
- which choices most often worked
- which doctrines emerged
- how the system is changing over time

This is a system-level reflection layer, not a control layer.
It must remain deterministic, advisory, and read-only.

Create:
- src/audit/ecosystem_loop_types.ts
- src/audit/ecosystem_loop_summary.ts
- src/ui/components/audit/EcosystemLoopSummaryPanel.tsx

Requirements:

1. Add canonical types.

interface PredictionToDecisionPattern {
  label: string;
  count: number;
  summary: string;
}

interface GovernanceChoiceOutcomePattern {
  label: string;
  count: number;
  summary: string;
}

interface DoctrineEmergencePattern {
  heuristicId: string;
  title: string;
  supportCount: number;
  confidence: "low" | "moderate" | "high";
  summary: string;
}

interface EcosystemChangeSummary {
  stabilizing: boolean;
  drifting: boolean;
  overcorrecting: boolean;
  summaryLines: string[];
}

interface EcosystemLoopSummary {
  totalAuditRuns: number;
  totalPredictions: number;
  totalGovernanceActions: number;
  totalOutcomeReviews: number;

  predictionToDecisionPatterns: PredictionToDecisionPattern[];
  governanceChoiceOutcomePatterns: GovernanceChoiceOutcomePattern[];
  doctrineEmergencePatterns: DoctrineEmergencePattern[];

  ecosystemChangeSummary: EcosystemChangeSummary;

  summaryLines: string[];
}

2. Build deterministic functions:
- buildPredictionToDecisionPatterns(...)
- buildGovernanceChoiceOutcomePatterns(...)
- buildDoctrineEmergencePatterns(...)
- buildEcosystemChangeSummary(...)
- buildEcosystemLoopSummary(...)

3. Pattern extraction rules:

Prediction -> decision patterns:
- identify repeated cases where certain predicted variables or risk directions led to promote / rollback / hold

Governance choice -> outcome patterns:
- identify repeated cases where promote / rollback / hold led to met_expectations / partially_met / did_not_meet

Doctrine emergence:
- summarize playbook heuristics that repeatedly appear with meaningful support

System change summary:
- stabilizing if lawful outcomes rise, recurring violations fall, or governance outcomes improve
- drifting if recurring violations grow, governance outcomes worsen, or unresolved outcomes rise
- overcorrecting if rollback/promotion churn rises without clear improvement

4. Example summary lines:
- "Protein placement risk most often led to rollback decisions in nutrition."
- "Promotions following conservative recommendations most often met expectations."
- "A doctrine favoring cautious promotion under shallow history has emerged repeatedly."
- "Recent governance patterns suggest stabilization."
- "Recent policy churn suggests possible overcorrection."

5. UI:
Create EcosystemLoopSummaryPanel showing:
- total counts
- prediction -> decision patterns
- governance choice -> outcome patterns
- doctrine emergence
- change summary
- top-level summary lines

6. Keep everything deterministic.
7. Do not use LLM generation.
8. Do not generate policy actions from this layer.
9. This layer is reflective and descriptive only.

Success condition:
NOMOS can summarize the full governance ecosystem across time, showing how prediction, decision, outcome, and doctrine interact at system scale.
