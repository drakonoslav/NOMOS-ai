Yes — that is the right next step.

That is the final loop-closure layer.

Up to now, NOMOS can produce:
	•	a recommendation
	•	a governance brief
	•	a promotion or rollback
	•	a later outcome review

But this step makes it possible to say:

this decision brief caused this governance action, which later produced these measurable outcomes

That is what turns the whole thing into a controlled reasoning ecosystem.

What to tell Replit

Build decision → outcome linkage for NOMOS.

Goal:
Link each governance deliberation summary to the actual governance action taken and then to the later observed outcomes, so NOMOS can trace:

deliberation -> action -> expected gains -> actual outcomes

This closes the loop between prediction, governance, and learning.

Create:
- src/audit/decision_outcome_link_types.ts
- src/audit/decision_outcome_linkage.ts
- src/ui/components/audit/DecisionOutcomeLinkPanel.tsx

Requirements:

1. Add canonical types.

interface GovernanceDecisionRecord {
  decisionId: string;
  timestamp: string;

  domain: "nutrition" | "training" | "schedule" | "generic";

  deliberationSummaryId: string;
  currentPolicyVersionId: string | null;
  recommendedPolicyVersionId: string | null;

  decision: "promote" | "rollback" | "hold";
  chosenPolicyVersionId: string | null;

  expectedGains: string[];
  expectedTradeoffs: string[];
  expectedRisks: string[];

  humanReason: string;
}

interface DecisionOutcomeLink {
  decisionId: string;
  deliberationSummaryId: string;
  governanceActionId: string | null;
  governanceOutcomeReviewId: string | null;

  decision: "promote" | "rollback" | "hold";
  chosenPolicyVersionId: string | null;

  expectedGains: string[];
  expectedTradeoffs: string[];
  expectedRisks: string[];

  actualOutcomeClass:
    | "met_expectations"
    | "partially_met"
    | "did_not_meet"
    | "insufficient_followup"
    | null;

  actualOutcomeLines: string[];

  linkageSummaryLines: string[];
}

interface DecisionOutcomeLinkReport {
  totalLinkedDecisions: number;
  links: DecisionOutcomeLink[];
  summaryLines: string[];
}

2. Build deterministic functions:
- buildGovernanceDecisionRecord(deliberationSummary, humanDecision, humanReason)
- linkDecisionToGovernanceAction(decisionRecord, governanceAuditTrail)
- linkDecisionToOutcomeReview(decisionRecord, governanceOutcomeReviews)
- buildDecisionOutcomeLink(decisionRecord, governanceAuditTrail, governanceOutcomeReviews)
- buildDecisionOutcomeLinkReport(decisionRecords, governanceAuditTrail, governanceOutcomeReviews)

3. Linking rules:
- a deliberation summary may produce a governance decision
- a governance decision may produce a governance action
- a governance action may later have an outcome review
- all linking must be done by stable ids, not fuzzy matching

4. Summary lines should support statements like:
- "This deliberation led to promotion of policy v3."
- "Expected gains included improved exact-match performance."
- "Later outcome review classified the action as partially_met."
- "Observed outcome: aggressiveness fell, but unresolved outcomes increased."

5. UI:
Create DecisionOutcomeLinkPanel showing:
- deliberation summary reference
- chosen decision
- chosen policy
- expected gains / tradeoffs / risks
- linked governance action
- linked outcome review
- outcome class
- linkage summary lines

6. Keep everything deterministic.
7. Do not use LLM generation.
8. Do not infer missing links if ids are unavailable.
9. A hold decision should still be storable and linkable, even if no governance action follows.

Success condition:
NOMOS can trace a specific governance deliberation to the decision taken and then to the later observed outcome, preserving the full chain of accountability.

Why this matters

This gives NOMOS the ability to say things like:
	•	“This deliberation recommended promotion of v3.”
	•	“The human accepted that recommendation.”
	•	“The action was promotion.”
	•	“Expected gain was improved exact-match rate.”
	•	“Later review showed the action only partially met expectations.”

That is much stronger than just having isolated records.

What the panel should eventually show

A good linked record would read like this:

Deliberation
	•	Recommendation: Promote v3
	•	Confidence: Moderate
	•	Expected gains: exact-match improvement, lower aggressiveness

Decision
	•	Human decision: Promote
	•	Chosen policy: v3
	•	Reason: Better nutrition exact-match with acceptable tradeoffs

Outcome
	•	Review class: partially_met
	•	Actual outcomes:
	•	exact-match improved
	•	unresolved outcomes increased slightly

Linkage summary
	•	This deliberation led to promotion of v3.
	•	Expected exact-match gain was realized.
	•	Tradeoff in unresolved outcomes was larger than expected.

That is a complete chain.

The key discipline

This linkage layer must remain factual:
	•	what was recommended
	•	what was chosen
	•	what was expected
	•	what happened

It should not rewrite history or reinterpret prior records.

What comes after this

After decision → outcome linkage, the strongest next step is:

ecosystem loop summary

A top-level layer that can summarize, across the whole system:

That would give NOMOS a true system-level reflective dashboard.
