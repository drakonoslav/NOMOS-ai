Build decisive-variable trend tracking for NOMOS.

Goal:
Track decisive variables across many audit runs so NOMOS can identify:
- recurring degradation drivers
- repeated violations
- variable frequency
- variable streaks
- stabilization vs drift over time

Create:
- src/audit/trend_types.ts
- src/audit/decisive_variable_trends.ts
- src/ui/components/audit/DecisiveVariableTrendPanel.tsx

Requirements:

1. Add canonical trend types.

interface DecisiveVariableOccurrence {
  versionId: string;
  timestamp: string;
  candidateId?: string | null;
  overallStatus: string | null;
  decisiveVariable: string | null;
}

interface DecisiveVariableTrend {
  variable: string;
  count: number;
  firstSeen: string;
  lastSeen: string;
  currentStreak: number;
  longestStreak: number;
  statuses: Record<string, number>;
}

interface DriftSummary {
  mostFrequentVariable: string | null;
  mostRecentVariable: string | null;
  recurringViolations: string[];
  stabilizing: boolean;
  drifting: boolean;
  summaryLines: string[];
}

interface DecisiveVariableTrendReport {
  totalRuns: number;
  variables: DecisiveVariableTrend[];
  driftSummary: DriftSummary;
  occurrenceTimeline: DecisiveVariableOccurrence[];
}

2. Build deterministic functions:
- extractDecisiveVariableOccurrences(auditRecords)
- buildDecisiveVariableTrends(occurrences)
- computeCurrentStreak(variable, occurrences)
- computeLongestStreak(variable, occurrences)
- buildDriftSummary(trends, occurrences)
- buildDecisiveVariableTrendReport(auditRecords)

3. Trend rules:
- count frequency per decisive variable
- compute streaks based on chronological order
- ignore null decisive variables for frequency ranking
- support lawful runs with null or "none" decisive variable
- recurring violation = variable appears >= 2 times in recent history
- stabilizing = recent sequence shows decreasing violations or more lawful outcomes
- drifting = recent sequence shows repeated same violation or worsening outcomes

4. UI:
Create DecisiveVariableTrendPanel that shows:
- most frequent decisive variable
- most recent decisive variable
- recurring violations
- per-variable counts
- streak lengths
- simple chronological list of occurrences

5. Add readable summaries like:
- "Protein placement violation has recurred in 3 consecutive runs."
- "Calorie delta is the most frequent recent degradation driver."
- "Recent runs suggest stabilization toward lawful status."

6. Keep everything deterministic.
7. Do not use LLM generation here.

Success condition:
NOMOS can inspect audit history and report which decisive variables recur, which are trending, and whether the system is stabilizing or drifting.
