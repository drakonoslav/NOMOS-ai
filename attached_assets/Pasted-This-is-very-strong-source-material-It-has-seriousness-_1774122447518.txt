NOMOS — Constitutional AI Reasoning Ecosystem

NOMOS is a deterministic, constraint-based reasoning ecosystem built in TypeScript. It transforms unstructured input into constrained, evaluable declarations and enforces constitutional laws before evaluation is permitted.

It is not a chatbot and not a freeform planner. It is a structured reasoning system with immutable audit history, explicit governance boundaries, and traceable decision support.

Canonical Runtime

NOMOS has one canonical runtime path:
	•	Frontend: artifacts/nomos-dashboard
	•	Backend: artifacts/api-server
	•	Shared reasoning library: packages/constitutional-kernel

Non-canonical or non-runtime folders:
	•	Demo-only: artifacts/mockup-sandbox
	•	Support / archival materials: attached_assets/

The workspace root coordinates the dashboard and API server. The root must not boot directly into the kernel package.

Quick Start

pnpm install
pnpm run dev

Expected result:
	•	the dashboard opens as the main app
	•	the API server responds under /api/*
	•	the kernel is executed by the API server, not as a root app

Production Runtime Path

The authoritative runtime path is:

User
  -> nomos-dashboard
  -> api-server
  -> constitutional-kernel

Core routes:
	•	/api/healthz
	•	/api/nomos/state
	•	/api/nomos/query/parse
	•	/api/nomos/query/evaluate

	•	artifacts/nomos-dashboard — canonical UI and cockpit runtime

What NOMOS Is

NOMOS is a Constitutional AI Kernel and reasoning ecosystem that:
	•	extracts structured facts from raw input
	•	detects gaps between what is declared and what is required
	•	supports field-level repair of incomplete declarations
	•	serializes confirmed drafts into canonical stable artifacts
	•	evaluates only confirmed complete declarations against typed constraints
	•	persists immutable audit history for every confirmed decision
	•	tracks decisive variable trends and failure predictions
	•	produces policy recommendations and governance support summaries
	•	computes ecosystem health with formula-level traceability
	•	presents the system through a role-aware cockpit and guided workflows
	•	records session worklog and replay as deterministic operational trace

Four Constitutional Laws

Law	Invariant
I	NOMOS never evaluates raw input
II	Only a confirmed canonicalDeclaration may be submitted for evaluation
III	Governance is advisory until explicit human action
IV	Historical records, frozen policy snapshots, and audit records are immutable

End-to-End Pipeline

Raw Input (natural language)
    │
    ▼ Intent detection → domain template selection
    ▼ Field extraction → ExtractedFields
    ▼ Gap detection → missingRequired / isEvaluable
    ▼ Structured draft assembly → StructuredDraft
    ▼ User-confirmed field patching
    ▼ Canonical declaration serialization → canonicalDeclaration (frozen)
    ▼ Constraint evaluation → EvaluationReport + ConstraintTrace
    ▼ Audit record persistence → AuditRecord (immutable, version-linked)
    ▼ Decisive variable trend tracking → DecisiveVariableTrendReport
    ▼ Failure prediction → calibration → bounded adjustment
    ▼ Policy bench → recommendation → regime comparison
    ▼ Governance decision support → deliberation summary
    ▼ Human governance action (promote / rollback / hold)
    ▼ Governance audit record (immutable)
    ▼ Outcome review → learning summary → playbook extraction
    ▼ Decision → outcome linkage
    ▼ Ecosystem loop summary
    ▼ Ecosystem health index + traceability
    ▼ Ecosystem cockpit snapshot
    ▼ Role-aware cockpit UI
    ▼ Session worklog
    ▼ Session replay / narrative reconstruction

Kernel	packages/constitutional-kernel	deterministic reasoning library used by backend

Health Index

NOMOS computes a deterministic four-component health index:

Component	Weight	Summary
Stability	35%	failure rate + streak modifiers
Calibration quality	25%	exact/directional match scoring with penalties
Governance effectiveness	25%	met and partial-rate based scoring
Policy churn	15%	bounded churn score with drift modifier

Bands:
	•	poor: 0–24
	•	fragile: 25–49
	•	stable: 50–74
	•	strong: 75–100
