# Epistemic AI Demo

A runnable TypeScript scaffold for the small constitutional kernel demo.

## What is included

This repo contains a tiny end-to-end demo that wires together:

- `belief_state.ts`
- `observer.ts`
- `feasibility_engine.ts`
- `robustness_analyzer.ts`
- `verification_kernel.ts`
- `model_registry.ts`
- `decision_engine.ts`
- `constitution_guard.ts`
- `audit_log.ts`
- `main.ts`

The runtime flow is:

`belief -> observer -> model -> feasibility -> robustness -> decision -> verification -> constitution guard -> audit`

## Expected behavior

The demo:

1. registers a primary and fallback mock model
2. initializes a bounded belief state
3. processes a delayed measurement through the observer
4. scores model confidence
5. evaluates two candidate plans
6. selects a plan by robustness first, cost second
7. verifies the result
8. applies, degrades, or refuses action
9. writes an audit record

## Notes

This is a first-pass scaffold, not a production control stack.

A few simplifications are intentional:

- the observer uses a minimal innovation correction instead of a full Kalman or particle filter
- the robustness analyzer uses a first-pass proxy instead of full reachability or Lyapunov analysis
- the mock mission system is deliberately tiny so the full chain is easy to inspect

## Next logical upgrades

- replace the observer correction with EKF/UKF or particle filtering
- replace sensitivity proxies with true singular-value or reachability analysis
- add an MPC controller module and route actuation through it
- add tests for stale-manifold invalidation and verification status transitions
