Best implementation pattern

Do not rely on randomness for these two demo states.

Create an explicit scenario selector.

Add a scenario type

export type DemoScenario =
  | "lawful_baseline"
  | "refused_infeasible";

Add a scenario loader

export interface DemoScenarioConfig {
  measurement: MeasurementSnapshot;
  resourcesAtObserve: number[];
  disturbancesAtObserve: number[];
  operatorHints: string[];
  forceDeterministicFallback: boolean;
}

Then make:

function buildScenarioConfig(scenario: DemoScenario): DemoScenarioConfig

How to make the lawful path actually survive

Right now your planner may screen everything out if margins are too tight.

For lawful_baseline, loosen them deliberately:
	•	increase fuel
	•	reduce disturbance
	•	reduce control aggressiveness
	•	reduce sensitivity matrix values
	•	widen terminal tolerance slightly
	•	keep dependency stamps identical

For example
	•	resourcesAtObserve = [1.0]
	•	disturbancesAtObserve = [0.02]
	•	conservative control sequence from fallback proposer
	•	sensitivity matrix:

[
  [1.05, 0.05],
  [0.02, 1.03]
]

	•	terminal condition:

Math.abs((x[0] ?? 0) - 1.0) - 0.95

That gives enough room for one candidate to survive.

⸻

How to make the refused path clean

For refused_infeasible, keep everything else normal and fail one hard thing.

Best options:
	1.	fuel below zero after rollout
	2.	stale solution stamp mismatch
	3.	impossible terminal bound

The cleanest is fuel/resource violation.

Example
	•	resourcesAtObserve = [0.04]
	•	same fallback control rollout burns > 0.04
	•	resourceChecks fail
	•	plan rejected
	•	no survivor
	•	final result = refused
