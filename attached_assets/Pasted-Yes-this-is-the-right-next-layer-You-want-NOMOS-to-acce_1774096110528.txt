What the auto-compiler should do

If the user types:

fix my meal plan

NOMOS should not guess a solution.

It should compile that into something like:

STATE:
- User wants a meal-plan audit/correction task.
- Meal structure is likely already defined but not yet parsed.
- Nutrition labels and phase targets may be relevant.

CONSTRAINTS:
- Unknown until clarified.
- Common likely constraints: preserve meal timing, preserve protein placement, keep calories close to target.

UNCERTAINTIES:
- No meal plan attached yet.
- No food label truth attached yet.
- No correction mode selected.

CANDIDATES:
A:
- Audit only.

B:
- Audit + minimal correction.

OBJECTIVE:
- Determine whether the meal system is accurate and whether correction is needed.

Then NOMOS should say:

Submission incomplete. Meal plan and source labels required.
Add the current meal system and attached food labels.

That’s the move.

⸻

The auto-compiler architecture

You want 5 stages:

1. Intent detector

Classify the raw input:
	•	meal audit
	•	schedule audit
	•	training audit
	•	freeform unknown

2. Domain template selector

Pick a skeleton:
	•	nutrition_audit_template
	•	training_audit_template
	•	schedule_template

3. Field extractor

Pull out what’s already present:
	•	meal plan?
	•	labels?
	•	targets?
	•	constraints?
	•	requested correction mode?

4. Gap detector

Mark what’s missing.

5. Structured prompt emitter

Generate:
	•	STATE
	•	CONSTRAINTS
	•	UNCERTAINTIES
	•	CANDIDATES
	•	OBJECTIVE

with blanks or inferred placeholders clearly marked

Minimal compiler types

If you want the core types first, here’s the shape:

type IntentType =
  | "NUTRITION_AUDIT"
  | "TRAINING_AUDIT"
  | "SCHEDULE_AUDIT"
  | "GENERIC_CONSTRAINT_TASK"
  | "UNKNOWN";

type StructuredDraft = {
  state: string[];
  constraints: string[];
  uncertainties: string[];
  candidates: { id: string; text: string }[];
  objective: string[];
  missingFields: string[];
  intent: IntentType;
};


⸻

Important design rule

The compiler is allowed to:
	•	infer structure
	•	infer likely domain
	•	infer likely missing sections

The compiler is not allowed to:
	•	invent facts
	•	fabricate labels
	•	silently assume constraints are satisfied
	•	evaluate before confirmation

That line is everything.
