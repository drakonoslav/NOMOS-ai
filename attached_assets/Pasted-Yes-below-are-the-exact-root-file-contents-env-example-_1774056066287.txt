Troubleshooting

Missing API key
If startup fails with a missing API key error:
	•	confirm OPENAI_API_KEY is set in Replit Secrets
	•	or confirm .env exists locally and contains the key

Model call fails
If the OpenAI request fails:
	•	NOMOS should fall back to deterministic proposals when enabled
	•	the system should continue through the lawful screening pipeline

TypeScript import issues
Make sure:
	•	package.json includes "type": "module"
	•	tsconfig.json uses module: "NodeNext" and moduleResolution: "NodeNext"

Minimal Startup Check

A healthy startup path should look like:
	1.	belief initialized
	2.	observer updates belief from measurement
	3.	model confidence scored
	4.	llm_proposer generates proposals
	5.	decision engine screens candidates
	6.	verification kernel sets status
	7.	constitution guard applies, degrades, or refuses
	8.	audit log records the outcome
