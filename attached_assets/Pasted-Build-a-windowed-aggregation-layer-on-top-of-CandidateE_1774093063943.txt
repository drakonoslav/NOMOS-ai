Build a windowed aggregation layer on top of CandidateEvent so NOMOS can compute derived variables such as fast_carbs_grams_within_90min.

Goal:
Given structured CandidateEvent objects, compute reusable derived variables by:
- filtering events by domain/type/tags
- filtering by time window relative to a reference event
- summing quantities
- computing maxima/minima where needed

Create:
- src/evaluation/windowed_aggregator.ts

This layer should be generic and reusable, not specific to only one prompt.

==================================================
1. TYPES
==================================================

Use the existing CandidateEvent type.

Add these types:

export interface EventWindowFilter {
  /**
   * Reference anchor, e.g. "lifting"
   */
  referenceEvent?: string;

  /**
   * Inclusive lower bound in minutes relative to reference.
   * Example: -90 means 90 min before
   */
  minOffsetMinutes?: number;

  /**
   * Inclusive upper bound in minutes relative to reference.
   * Example: 0 means up to the moment of lifting
   */
  maxOffsetMinutes?: number;
}

export interface EventSelector {
  domain?: string;
  action?: string;
  subject?: string;
  nutrientSpeed?: "FAST" | "SLOW" | "UNKNOWN";
  requiredTags?: string[];
}

export interface AggregationResult {
  key: string;
  label: string;
  value: number;
  units: string;
  matchedEventIds: string[];
  notes: string[];
}

- selector filtering should match all specified fields

- quantity summation should ignore events with undefined quantity
- always return matchedEventIds and notes

==================================================
3. WINDOW FILTER SEMANTICS
==================================================

Use relative offsets in minutes.

Examples:
- 30 minutes before lifting => timeOffsetMinutes = -30
- 2 hours before lifting => timeOffsetMinutes = -120

For pre-lift windows:
- "within 90 minutes before lifting" means offsets in [-90, 0]
- "within 60 minutes before lifting" means offsets in [-60, 0]

So:
filterEventsByWindow(events, {
  referenceEvent: "lifting",
  minOffsetMinutes: -90,
  maxOffsetMinutes: 0
})

must include:
- -75
- -45
- -30

and exclude:
- -120

==================================================
4. GENERIC FILTER RULES
==================================================

Implement filterEventsBySelector with these semantics:

- domain exact match if specified
- action exact match if specified
- subject exact match if specified (case-insensitive)
- nutrientSpeed exact match if specified
- requiredTags => all required tags must be contained in event.tags

Case-insensitive matching for string comparisons.
