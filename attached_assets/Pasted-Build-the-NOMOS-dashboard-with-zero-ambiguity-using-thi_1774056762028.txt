Build the NOMOS dashboard with zero ambiguity using this exact layout and hierarchy.

- severe
- elegant
- verification-first
- dark theme default
- no chatbot UI
- no consumer SaaS softness

- / -> OverviewPage
- /belief -> BeliefPage
- /proposals -> ProposalsPage
- /decision -> DecisionPage
- /verification -> VerificationPage
- /audit -> AuditPage

App shell:
- fixed left sidebar
- top status bar
- main content area

Sidebar content:
- NOMOS logo
- title: NOMOS

- footer: active model, theme toggle

- page title

- authority badge

Data rule:
- wire UI to actual NOMOS runtime data if possible
- if not, create a typed dashboard state adapter from the existing main.ts outputs

Visual rules:
- dark theme default
- geometric panels
- low border radius
- strong borders
- no playful colors
- no gradients unless extremely restrained
- use status colors only for lawful / degraded / invalid
- use mono or semi-mono for vectors, IDs, and structured values

Critical architecture rule:
- Do not move authority out of verification_kernel or constitution_guard
- The UI displays state; it does not determine legality
