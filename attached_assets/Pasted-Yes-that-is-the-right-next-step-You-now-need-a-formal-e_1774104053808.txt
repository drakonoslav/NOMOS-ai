The invariants

This is the most important part. Have Replit enforce these in one place.

export function assertEvaluationReportInvariants(
  report: OverallEvaluationReport
): void {
  for (const candidate of report.candidates) {
    if (
      candidate.constraintsViolated > 0 &&
      candidate.summaryReason.toLowerCase().includes("all constraints passed")
    ) {
      throw new Error(
        `Invariant failed: candidate ${candidate.candidateId} has violated constraints but says all constraints passed.`
      );
    }

    if (
      candidate.constraintsInterpretationRequired === 0 &&
      candidate.summaryReason.toLowerCase().includes("manual review")
    ) {
      throw new Error(
        `Invariant failed: candidate ${candidate.candidateId} mentions manual review with no interpretation-required constraints.`
      );
    }

    if (
      candidate.decisiveVariable?.toLowerCase().includes("violation") &&
      candidate.constraintsViolated === 0
    ) {
      throw new Error(
        `Invariant failed: candidate ${candidate.candidateId} names a violation without any violated constraints.`
      );
    }

    if (
      candidate.verdict === "lawful" &&
      candidate.constraintsViolated > 0
    ) {
      throw new Error(
        `Invariant failed: candidate ${candidate.candidateId} is lawful but has violated constraints.`
      );
    }
  }
}

UI rules

The UI must render from fields, not prose guesses.

Tell Replit these exact rendering rules:
	•	“All constraints were evaluated deterministically.” only if constraintsInterpretationRequired === 0
	•	“All constraints passed.” only if constraintsViolated === 0 and constraintsNotEvaluated === 0
	•	“Violations detected.” if constraintsViolated > 0
	•	“Manual review required.” only if constraintsInterpretationRequired > 0

So for your current case, the correct text would be:
	•	“All constraints were evaluated deterministically.”
	•	“1 or more constraint violations detected.”

Not “all constraints passed.”

Decisive variable rule

The decisive variable must come from the highest-priority violated constraint, not from fallback text.

Priority suggestion:
	1.	violated structural lock
	2.	violated source truth
	3.	violated allowed action
	4.	target tolerance miss
	5.	interpretation required
