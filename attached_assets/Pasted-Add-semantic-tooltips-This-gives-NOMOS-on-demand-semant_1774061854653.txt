Add semantic tooltips

This gives NOMOS on-demand semantic depth without polluting the interface.

visible layer = minimal
hover layer = precise meaning

⸻

🧠 What we’re building

When user hovers:
	•	robustness margin → definition + current value context
	•	feasibility constraint → exact violated rule
	•	model confidence → what it represents
	•	observability → why it matters

⸻

🧱 1. Create semantic dictionary

📁 src/ui/semantic/semantic_map.ts

export interface SemanticEntry {
  label: string;
  description: string;
  dynamic?: (ctx: any) => string;
}

export const SEMANTIC_MAP: Record<string, SemanticEntry> = {
  "robustness margin": {
    label: "Robustness Margin",
    description:
      "Distance between current operating point and failure threshold.",
    dynamic: (ctx) =>
      ctx?.robustnessEpsilon !== undefined
        ? `ε = ${ctx.robustnessEpsilon} (min ${ctx.robustnessEpsilonMin})`
        : "",
  },

  "feasibility constraint": {
    label: "Feasibility Constraint",
    description:
      "A required condition that must be satisfied for any valid action.",
    dynamic: (ctx) =>
      ctx?.activeConstraint
        ? `Violated: ${ctx.activeConstraint}`
        : "",
  },

  "model confidence": {
    label: "Model Confidence",
    description:
      "Estimated reliability of the system model under current conditions.",
    dynamic: (ctx) =>
      ctx?.modelConfidence !== undefined
        ? `Confidence: ${ctx.modelConfidence.toFixed(2)}`
        : "",
  },

  observability: {
    label: "Observability",
    description:
      "Ability to infer internal system state from available measurements.",
  },

  identifiability: {
    label: "Identifiability",
    description:
      "Ability to uniquely determine system parameters from data.",
  },
};

🧩 2. Tooltip component

📁 SemanticTooltip.tsx

import React, { useState } from "react";
import { SEMANTIC_MAP } from "../semantic/semantic_map";

export function SemanticTooltip({
  term,
  context,
  children,
}: {
  term: string;
  context?: any;
  children: React.ReactNode;
}) {
  const [hover, setHover] = useState(false);

  const entry = SEMANTIC_MAP[term.toLowerCase()];
  if (!entry) return <>{children}</>;

  return (
    <span
      className="nm-tooltip-wrapper"
      onMouseEnter={() => setHover(true)}
      onMouseLeave={() => setHover(false)}
    >
      {children}

      {hover && (
        <div className="nm-tooltip">
          <div className="nm-tooltip-title">{entry.label}</div>
          <div className="nm-tooltip-desc">
            {entry.description}
          </div>

          {entry.dynamic && (
            <div className="nm-tooltip-dynamic">
              {entry.dynamic(context)}
            </div>
          )}
        </div>
      )}
    </span>
  );
}

🧠 Why this is powerful

Without tooltips:
	•	system feels rigid
	•	terms feel opaque

With tooltips:
	•	system remains minimal
	•	depth is available only when needed

⸻

🧠 Design integrity preserved

You did NOT:
	•	add labels everywhere
	•	clutter UI
	•	break flow

You DID:
	•	attach meaning to structure
	•	keep interaction optional
	•	preserve system tone
