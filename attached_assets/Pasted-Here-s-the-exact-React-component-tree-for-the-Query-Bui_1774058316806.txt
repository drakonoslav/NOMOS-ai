Component responsibilities

QueryBuilderPage

Owns:
	•	mode (guided or natural)
	•	local draft state
	•	parsed NomosQuery
	•	submission quality
	•	evaluate action

Suggested shape:

type QueryMode = "guided" | "natural";

interface QueryBuilderPageState {
  mode: QueryMode;
  rawInput: string;
  guidedDraft: GuidedQueryDraft;
  parsedQuery?: NomosQuery;
  parseErrors: string[];
  isParsing: boolean;
  isEvaluating: boolean;
}

Field rules

SituationField
Single multiline text area.
Prompt:

Describe the current situation in plain terms.

FactsListEditor
Array of short fact entries.
Examples:
	•	“Current job stable”
	•	“Fuel remaining is low”
	•	“Sensor delay is 40ms”

ConstraintsListEditor
Array of hard limits.
Examples:
	•	“Must avoid income gap > 2 months”
	•	“Fuel must remain non-negative”

UncertaintiesListEditor
Array of unknowns.
Examples:
	•	“Culture at new job uncertain”
	•	“Disturbance magnitude may rise”

Recommended layout proportions

Desktop:
	•	left input column: 56%
	•	right preview column: 44%

Tablet:
	•	stacked
	•	input first, preview second

Mobile:
	•	one column
	•	preview under actions

Interaction rules

Guided mode
	•	updates local draft only
	•	parse happens on button press or debounced if desired

Natural language mode
	•	parse on button press
	•	later can upgrade to live parse preview

Evaluation rule

Do not let the page evaluate raw text directly.

Always require:
	1.	parse
	2.	preview
	3.	confirm evaluate

That preserves NOMOS identity.

Visual tone

This page should feel:
	•	structured
	•	serious
	•	not like a chat window

So:
	•	no chat bubbles
	•	no assistant avatar
	•	no “Ask NOMOS anything”

Instead use:
	•	Submit Query
	•	Parse Submission
	•	Evaluate Candidates
	•	Preview Structured Query
