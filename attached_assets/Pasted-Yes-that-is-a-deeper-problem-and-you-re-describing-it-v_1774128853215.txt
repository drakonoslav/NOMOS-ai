What is probably happening

Very likely one or more of these:

1. Guided mode writes structured fields directly

So it may bypass parts of the real compiler.

2. Natural language mode goes through parser heuristics

So it depends on extraction logic.

3. Auto-compile mode uses template + gap filling + inference

So it may normalize differently than the other two.

That means you don’t actually have:
	•	one canonical compiler with three front doors

You may have:
	•	three different roads that only partially meet downstream

That’s why they “don’t parse the same reality.”

The correct architecture

You want this:

Guided input --------\
Natural language ----- > Unified compiler pipeline -> Canonical declaration
Auto-compile --------/

Not this:

Guided -> special logic -> declaration A
NL -> parser logic -> declaration B
Auto -> template logic -> declaration C

The first is coherent.
The second guarantees drift.

Guided mode

	•	field collection

Natural Language mode

Auto-Compile mode

So the compiler core is shared, and only the input acquisition layer differs.
