Requirements:

1. Add canonical types.

interface HealthComponentTrace {
  component: "stability" | "calibrationQuality" | "governanceEffectiveness" | "policyChurn";

  rawInputs: Record<string, number | string | boolean | null>;
  formulaLines: string[];
  weightedContribution: number;

  contributingRecordIds: string[];
  explanationLines: string[];
}

interface EcosystemHealthTrace {
  overallFormulaLines: string[];
  overallInputs: Record<string, number>;

  componentTraces: HealthComponentTrace[];
}

2. Build deterministic functions:
- buildStabilityTrace(...)
- buildCalibrationQualityTrace(...)
- buildGovernanceEffectivenessTrace(...)
- buildPolicyChurnTrace(...)
- buildEcosystemHealthTrace(...)

3. Each component trace must show:
- exact raw input values
- exact score formula used
- final component score
- record ids or action ids that contributed

4. Overall trace must show:
- component scores
- weight per component
- final weighted calculation

5. Example formula lines:
- "stability = clamp(0.5 * lawfulShare + 0.3 * driftReduction + 0.2 * recurringViolationReduction, 0, 100)"

6. UI:
- EcosystemHealthPanel should allow clicking:
  - overall score
  - each component score
- opening a component shows:
  - raw inputs
  - formula
  - contributing record ids
  - explanation lines

7. Keep everything deterministic.
8. Do not use LLM generation.
9. Do not hide missing inputs; show them explicitly if a component is weakened by shallow evidence.

Every health score in NOMOS can be traced back to exact inputs, explicit formulas, and contributing records.
