NOMOS Dashboard Specification

Primary design goal

The UI must communicate this instantly:

NOMOS is not here to help. It is here to decide whether action is lawful.

So the dashboard should read like:
	•	a control room
	•	a constitutional instrument panel
	•	a systems verification console

Primitive rules

Panel

Base rectangular container for all system blocks.

Style:
	•	sharp corners or very low radius (6px max)
	•	strong border
	•	no soft drop shadows
	•	layered surface contrast

MetricCard

For single important values.

Used for:
	•	epsilonX
	•	proposal count
	•	model confidence
	•	robustness epsilon
	•	selected plan

StatusBadge

Only saturated element category.

States:
	•	lawful = muted deep green
	•	degraded = amber
	•	invalid = dark red
	•	authorized = steel blue or white-outline
	•	refused = red-outline

DataTable

Dense but breathable, no playful striping.

CodeBlock

For exact JSON-like or vector display.

11. Visual design rules

Surfaces

Three layers only:
	•	background
	•	panel
	•	elevated panel

Borders

Use borders, not blur-heavy shadows.

Radius

Low radius:
	•	4px to 8px max

Typography
	•	page titles: strong serif or severe display sans
	•	data: mono or semi-mono option for vectors and ids
	•	labels: uppercase/small caps

Spacing

Generous spacing. Do not cram.
This should feel judicial and engineered.

Charts

Minimal.
Prefer:
	•	bars
	•	line traces
	•	matrices
Do not use playful donut charts.
