# Example: Ecosystem Health Index + Cockpit Snapshot

This example shows the ecosystem health index computation and the resulting
cockpit snapshot with attention alerts.

## Files in This Example

| File | Stage | Description |
|------|-------|-------------|
| `health_index.json` | Health | EcosystemHealthIndex — 4-component weighted score |
| `health_trace.json` | Traceability | EcosystemHealthTrace — per-component formula + source record IDs |
| `cockpit_snapshot.json` | Cockpit | EcosystemCockpitSnapshot — all 7 blocks |
| `attention_alerts.json` | Attention | Extracted attention alerts with severity and source |

## Health Index Scoring

The health index is a weighted composite of four components:

| Component | Weight | Score (this example) | Band contribution |
|-----------|--------|---------------------|-------------------|
| Stability | 35% | 61 | 21.35 |
| Calibration quality | 25% | 72 | 18.00 |
| Governance effectiveness | 25% | 80 | 20.00 |
| Policy churn | 15% | 55 | 8.25 |
| **Total** | **100%** | **67.60** | **→ stable** |

## Traceability

Every component score is traceable to:
- the formula used to compute it
- the source record IDs that contributed to the inputs
- the weight applied

This means every cockpit health reading can be verified against its source records.

## Attention Alerts

The cockpit attention block is generated by 7 deterministic rules:

| Rule | Threshold | Triggered in this example? |
|------|-----------|---------------------------|
| Health component poor | score < 25 | No |
| Health component fragile | score < 50 | No |
| Low prediction confidence | confidence < 0.5 | No |
| Rising risk direction | riskDirection = "rising" | Yes |
| Recurring violation streak | streak ≥ 3 | No |
| Governance overcorrection | driftState = "overcorrecting" | No |
| Cautions exceed supporting doctrine | cautiousCount > supportingCount | No |
| High unresolved prediction rate | unresolvedRate > 0.30 | No |

## Key Invariant

The cockpit snapshot is a **read-only composition** of all underlying subsystems.
It does not modify any source data. The same snapshot can be derived from the same
inputs at any point in time with identical results.
